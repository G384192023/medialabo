「Webコンテンツ作成」の成果

本テーマについて

本授業は情報工学の基礎を応用し、実験、報告、協働作業、モーション解析、Web制作を通じて実践力と課題解決力を養うことを目的とし、グループワークやプレゼンテーションを通じて、リーダーシップやコミュニケーション能力を高めます。

プロジェクト課題について

プロジェクト課題では、ユーザーがジャンルを選択しグルメ情報を検索できるWebページを作成した。HTMLではselect要素とbuttonを設置し、JavaScriptでユーザーの選択値を取得。Axiosを利用してAPIからデータを非同期で取得し、選択されたジャンルに合致する店舗情報をフィルタリングした。検索結果の件数を画面上に表示し、各店舗の詳細情報をulとli要素で動的に生成してdiv#result内に表示した。sub_genreが存在しない場合に備え、条件演算子でエラー回避を行った。さらに、CSSで青系の複数色を用いて視認性とデザイン性を高めた。

プロジェクト課題で作成したページ: "project.html"

数当てゲーム

数当てゲームの説明:この数当てゲームは、1〜10の範囲でランダムに選ばれた1つの整数を正解の数として設定し、ユーザが3回以内にその数を予想、入力して当てることを目的としたゲームです。HTMLでは入力欄とボタン、予想回数や結果を表示する領域を用意し、JavaScriptでは乱数生成、予想の判定、結果の表示処理を行います。ユーザが数を入力してボタンを押すと、現在の予想回数と入力した数が表示され、正解なら「正解です.おめでとう!」、間違えなら答えが大きいか小さいかのヒントが出ます。3回目で外れた場合は「まちがい.残念でした答えは○です」と表示されます。すでに3回以上入力された後は、それ以降の入力は無視され「すでにゲームは終わっています」と表示される仕組みです。

数当てゲームのページ: kazuate.html

各授業回の学習事項

第1回 WWW と HTML

学習したこと:HTMLの基本について学びました。要素の構造や、ブロック要素とインライン要素の違い、空要素、属性の書き方と役割、主要なタグの使い方を理解しました。 また、実際にエディタを使ってHTMLファイルを作成し、簡単なWebページを作成する実践も行いました。

演習の成果: 演習1-4:HTMLのさまざまな要素

第2回 CSS によるWebページの装飾

学習したこと:HTMLの基本に加えて、CSSによるWebページの装飾方法も学びました。要素の色や文字サイズの変更、レイアウト調整、リンクのホバー時など状態に応じた装飾の方法を理解しました。Webページの見た目を整えるためにCSSが果たす役割や、HTMLと組み合わせて使う際の基本的な書き方・適用方法も学習しました。

演習の成果: 演習2-3:ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

学習したこと:JavaScriptの基本について学びました。Webページに動きを加えるために使われ、ボタン操作や位置情報取得、地図表示、アニメーション表示などが可能で、DOMやGeolocationなどのAPIも使って、HTMLやCSSの操作や地図表示など多彩な機能の実現ができます。

演習の成果: 演習3-6:オブジェクトの参照

第4回 DOM と JavaScript

学習したこと:JavaScriptによるイベント処理とDOMの基本を学びました。input要素の使い方、要素の検索、テキストや属性の取得・変更方法を実践し、HTMLでフォームを作り、JavaScriptでDOMを操作するなど、動的にWebページを操作する方法を理解しました。ユーザの操作に応じたWebページ作成が可能になります。

演習の成果: 演習4-10:足し算ページ

第5回 イベント処理

学習したこと:JavaScriptによるDOM操作の基本に加え、イベントリスナーを使ってユーザーの操作に応じて動的にHTMLを変更する方法やボタンで新しい要素を追加したり、フォーム入力に応じて内容を更新・削除する処理を学びました。実践的なスキルを通じて、Webページ開発の幅がさらに広がりました。

演習の成果: 演習5-7:DOMプログラミング(2)

工夫点,苦心点,強調点,感想

・工夫点

コードの見やすさを意識し、ページ全体の構成がわかりやすくなるよう工夫しました。HTMLでは、見出しタグやセクション分けを適切に使い、情報の区切りを明確にすることで、ユーザーがどこに何があるか迷わない構造を心がけました。CSSでは青系の色を中心に配色し、視認性とデザイン性のバランスをとるように工夫しました。見やすさに加え、飽きのこないシンプルなデザインを目指しました。こうした工夫を通じて、実際に動くWebページを制作する中で、機能だけでなくUIにも配慮する大切さを実感しました。

・苦心点

プログラム作成において、数当てゲームが初めての難所だと感じた制作だったので、何から手をつければいいのか分からず、最初はとても苦労しました。特に、正解かどうかの判定や、残り回数の管理、途中で正解した場合にそれ以上の入力を無効にする処理など、条件分岐の組み立てに時間がかかりました。どこで何を判定するべきか、どの順番で処理を書くべきかを考えるのが難しく、何度も試行錯誤を繰り返しました。また、プロジェクト課題で使った function print()と printDom() のように似ている関数に混乱しました。それぞれがどこで使われ、何をしているのかを丁寧に確認しながら、ようやく役割を正しく理解し、使い分けられるようになりました。

・強調点

実際の店舗情報を用いてデータを構造的に表示する際、単純なリスト形式ではなく、段落やラベルを活用することで、情報がより読みやすくなるようにしました。具体的には、住所、営業時間、ジャンルなどの各項目を整理し、ユーザーが直感的に把握しやすいように表示を整えました。

・感想

初めは「Webページを作る」と言われても、正直どこから手をつければいいのか分からず、不安な気持ちが強かったです。ですが、授業を通してHTML・CSS・JavaScriptの基本から一つ一つ学ぶことができ、最終的には自分なりの形に仕上げることができたので、大きな達成感を得ました。特に感じたのは、オリジナリティを出すことの難しさです。単に動くものを作るだけでなく、「他と違う工夫」や「自分らしさ」を出そうとすると、何をどう変えれば良いのか悩む場面が多くありました。また、デザインセンスも問われるので、文字の大きさや色のバランス、背景との見やすさなどを調整するのに何度も試行錯誤しました。 JavaScriptでは、ただ正しいコードを書くのではなく、どこから考え始めるか、どんな関数をどう組み立てるかといった、設計の段階が特に難しかったです。今回の経験を通して、Web制作の楽しさと奥深さの両方を実感しました。今後はより見やすく、使いやすいWebページを作ることを目指して、また学ぶ時があればさらに学びを深めていきたいです。
