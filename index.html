数当てゲーム

数当てゲームの説明:この数当てゲームは、1〜10の乱数を1つ決めて答えとし、ユーザが3回以内に答えの数を予想し、入力して当てるゲームです。HTMLでは入力欄とボタン、予想回数や結果を表示する領域を用意し、JavaScriptでは乱数生成、予想の判定、結果の表示処理を行います。ユーザが数を入力してボタンを押すと、現在の予想回数と入力した数が表示され、正解なら「正解です.おめでとう！」、間違えなら答えが大きいか小さいかのヒントが出ます。3回目で外れた場合は「まちがい.残念でした答えは○です」と表示されます。すでに3回以上入力された後は、それ以降の入力は無視され「すでにゲームは終わっています」と表示される仕組みです。

各授業回の学習事項

第1回 WWW と HTML

学習したこと:HTMLの基本について学びました。要素の構造や、ブロック要素とインライン要素の違い、空要素、属性の書き方と役割、主要なタグの使い方を理解しました。 また、実際にエディタを使ってHTMLファイルを作成し、簡単なWebページを作成する実践も行いました。

演習の成果： 演習1-4：HTMLのさまざまな要素

第2回 CSS によるWebページの装飾

学習したこと:HTMLの基本に加えて、CSSによるWebページの装飾方法も学びました。要素の色や文字サイズの変更、レイアウト調整、リンクのホバー時など状態に応じた装飾の方法を理解しました。Webページの見た目を整えるためにCSSが果たす役割や、HTMLと組み合わせて使う際の基本的な書き方・適用方法も学習しました。

演習の成果： 演習2-3：ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

学習したこと:JavaScriptの基本について学びました。Webページに動きを加えるために使われ、ボタン操作や位置情報取得、地図表示、アニメーション表示などが可能で、DOMやGeolocationなどのAPIも使って、HTMLやCSSの操作や地図表示など多彩な機能の実現ができます。

演習の成果： 演習3-6：オブジェクトの参照

第4回 DOM と JavaScript

学習したこと:JavaScriptによるイベント処理とDOMの基本を学びました。input要素の使い方、要素の検索、テキストや属性の取得・変更方法を実践し、HTMLでフォームを作り、JavaScriptでDOMを操作するなど、動的にWebページを操作する方法を理解しました。ユーザの操作に応じたWebページ作成が可能になります。

演習の成果： 演習4-10：足し算ページ

第5回 イベント処理

学習したこと:JavaScriptによるDOM操作の基本に加え、イベントリスナーを使ってユーザーの操作に応じて動的にHTMLを変更する方法やボタンで新しい要素を追加したり、フォーム入力に応じて内容を更新・削除する処理を学びました。実践的なスキルを通じて、Webページ開発の幅がさらに広がりました。

演習の成果： 演習5-7：DOMプログラミング(2)

工夫点,苦心点,強調点,感想

・工夫点 1.HTMLの構造設計 ページ全体の構成を読みやすくするために、見出しタグ（

〜

・苦心点 1.HTMLの階層構造と見出しの使い方 適切なタグ（例：

、
など）を選ぶのに迷いました。特に以降が存在しないことに気づかず、表示されない問題に悩みました。最終的には正しいタグ階層を学んで、見直しました。 2.JavaScriptでの条件分岐 予想が正解かどうか、または残り回数があるかをチェックするロジックの実装でつまずきました。特に、3回で終わる条件や、途中で正解した場合にそれ以上の入力を無効にする処理をどう書くかに苦労しました。 3.CSSの装飾バランス 文字の大きさや色の調整で、見やすさとデザイン性のバランスを取るのが難しかったです。背景色やフォント色が見づらくなってしまったため、何度も試行錯誤しました。

・強調点 1.プロジェクト課題での実践力 実際の店舗情報を使ってデータを構造的に表示する部分では、リスト形式ではなく段落やラベルを使って、より読みやすい表示になるよう意識しました。住所・営業時間・ジャンルなどの情報を整理し、見やすくまとめました。 2.数当てゲームの完成度 単に「当たった・外れた」だけでなく、ユーザーに対するヒント（「大きい」「小さい」）を追加し、ゲームとしての楽しさを高めました。また、連続での再プレイを防ぐ処理を入れたことで、バグを防ぎ、意図しない操作への耐性も持たせました。 3.学習の積み重ねの成果 1回目から5回目までの授業で学んだ内容を、すべて自分なりに反映できたことが強みです。HTML/CSS/JavaScriptのそれぞれの役割を意識して使い分けられるようになりました。

・感想 初めは「Webページを作る」と言っても何から始めればいいかわからず不安でしたが、授業を通して基礎から段階的に学べたことで、最終的には1つのページとして形にすることができ、自信になりました。 特にJavaScriptの学習では、動作を確認しながらプログラムを書く体験がとても新鮮で、コードの一部を変更するだけで結果が大きく変わることに驚きと面白さを感じました。 今後は、今回学んだ技術を応用して、もっとインタラクティブでデザイン性の高いWebページにも挑戦してみたいと思います。また、今回のように「ユーザー目線での使いやすさ」を考えることが、Web制作においてとても大切だと実感しました。
